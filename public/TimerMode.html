STEG 3 · Figma-design · UX-beslutninger

Designvalg og begrunnelser KM: UI/UX-design

Tidsvalg: 30s / 60s / 120s

30s = rask øvelse uten press · 60s = standard for de fleste · 120s = konsentrert øvelse

Flow mode — ikke Strict mode

Strict mode stopper ved feil. Under tidspress er det frustrerende og ødelegger rytmen.

Nedtelling øverst til høyre

Synlig men ikke forstyrrende. Rød farge under 10 sekunder som visuell advarsel.

Back → samme øvingstekst

Returnerer til samme tekst, ikke forsiden — brukeren mister ikke fremdriften sin.

STEG 4 · Implementering · Prompt til Claude Code

Prompten jeg sendte til Claude Code

prompt.md
I'm adding a Timer mode to my existing multilingual typing practice site called Polyglotyping, built with Astro and SCSS. ## Current state The "Typing practice" section has two mode buttons: "Strict mode" and "Flow mode". There is also a Progress bar, Accuracy display, and WPM counter. ## What I want to add ### 1. Timer mode button - Add a third button "Timer mode" next to the existing buttons - When selected, it should appear highlighted/active ### 2. Time selection (only visible when Timer mode is selected) - Show three time buttons: "30s" | "60s" | "120s" - Default selected time should be 60s - Timer mode runs in Flow mode ### 3. Countdown display - Show a countdown timer when the user starts typing - Update every 1 second (1000ms interval) - Display as a small circle in the top-right corner of the text area - Show only remaining seconds (e.g. 60, 59, 58...) - Change color to red or orange under 10 seconds ### 4. Results screen (when time runs out) - Replace "Finished" with "Time's up!" - Keep Accuracy, WPM, and Mistakes cards - Remove "How well did you understand this text?" section - Replace with three buttons: "Try again" | "Choose new time" | "Back" ## IMPORTANT – do not break existing functionality - Do NOT modify any existing functionality - Do NOT change Strict mode or Flow mode behavior - Do NOT remove or rename any existing CSS classes - Do NOT change any existing SCSS variables - Only add new code, do not delete existing code ## File to edit - Main file: src/pages/review.astro - Styles: src/styles/global.scss

Hva prompten inneholder

  • Navaerende tilstand beskrevet
  • Hva som skal legges til (4 punkter)
  • Design requirements
  • IMPORTANT: hva som IKKE skal endres
  • Hvilke filer som skal redigeres
